Implementation Notes (Tenant Guide Reference)

Two-Step Confirmation Pattern (LOCKED - Section 1.4):

  1. Step 1: User selects a choice (F/L/R or 1/2/3)
  2. Step 2: User confirms with Enter or clicks Submit button
  3. Submit button disabled until choice selected (opacity-50 cursor-not-allowed)
  4. Submit button activates with pulse animation when choice made
  5. Selected choice shows: scale-105 ring-2 ring-{color}-400 shadow-xl
  6. Unselected choices dim: opacity-60

Keyboard Shortcuts (LOCKED - Section 1.4):

  • F/L/R (or semantic letters) - Select choice
  • 1/2/3 - Fallback numeric selection
  • Enter - Submit selected choice
  • S - Skip prediction (always available)

Modal Size (LOCKED - Section 1.1):

className="max-w-lg w-full p-6"

No height constraint - content fits naturally with ‚â§3 choices limit

Grid Layout for Choices:

2 choices: grid-cols-2
3 choices: grid-cols-3

Button States:

  • Default: bg-{color}-600
  • Hover: hover:bg-{color}-500 hover:scale-105
  • Selected: scale-105 ring-2 ring-{color}-400 shadow-xl
  • Unselected (after selection): opacity-60
  • Submit disabled: opacity-50 cursor-not-allowed
  • Submit active: animate-pulse hover:scale-105

Why These Colors?

  • Emerald: Success/positive outcomes (found, keep, yes)
  • Orange: Discard/negative outcomes (covered, no, stop)
  • Blue: Left/backward navigation + Submit button (universal convention)
  • Red: Right/forward navigation (universal convention)
  • All have sufficient contrast with dark background (WCAG AAA)

Why Two-Step Confirmation?

  • Prevents accidental submissions (especially with keyboard shortcuts)
  • Gives user time to review their choice
  • Clear visual feedback before committing
  • Consistent with form design best practices
