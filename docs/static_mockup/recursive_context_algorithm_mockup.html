About Interval Coverage

The Interval Coverage algorithm (often related to Greedy Interval Scheduling) aims to find the minimum number of intervals needed to cover a target range, or to identify redundant intervals.

How it works:

  • Sort intervals by their start time.
  • Iterate through intervals while maintaining a max_end_so_far.
  • If a new interval's end is less than or equal to the current max, it is considered "covered" or redundant.
  • If it extends beyond the current max, we update our coverage.
